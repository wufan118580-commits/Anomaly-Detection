1.检查SSH密钥配置
# 查看是否已有 SSH 密钥
ls -la ~/.ssh

# 如果没有，生成新的 SSH 密钥
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# 查看公钥并添加到 GitHub
cat ~/.ssh/id_rsa.pub

然后将公钥添加到 GitHub：

登录 GitHub

Settings → SSH and GPG keys → New SSH key

粘贴公钥内容

2.测试 SSH 连接
ssh -T git@github.com

如果配置正确，会显示：Hi username! You've successfully authenticated...

3. 使用 HTTPS 替代 SSH
git clone https://github.com/wufan118580-commits/ai-learn.git

4.检查网络连接
# 测试 GitHub 连通性
ping github.com
# 或
curl -v https://github.com

5. 配置 SSH 客户端
# 配置 SSH 使用 443 端口（绕过防火墙）
echo "Host github.com
  Hostname ssh.github.com
  Port 443" >> ~/.ssh/config

# 测试 SSH 连接
ssh -T git@github.com

6.手动添加remote
# 添加 remote
git remote add origin https://github.com/wufan118580-commits/ai-learn.git

# 或者如果是 SSH
git remote add origin git@github.com:wufan118580-commits/ai-learn.git
git remote set-url origin git@github.com:wufan118580-commits/ai-learn.git

# 验证
git remote -v


# 分支操作
# 1. 先拉取远程仓库的最新信息（让本地知道远程有了新分支）
git fetch origin

# 2. 创建并切换到本地 develop 分支，同时与远程分支建立关联
git checkout -b develop origin/develop


# 查看当前分支（* 号所在的就是当前分支）
git branch

# 应该看到：
#   main
# * develop

# 查看本地分支和远程分支的跟踪关系
git branch -vv
# 应该看到类似：
# develop   abc1234 [origin/develop] 你的提交信息
